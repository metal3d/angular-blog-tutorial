Nous avons maintenant un controller pour créer un post de blog dans controllers/blogedit.js

Une vue a aussi été créée, dans views/blogedit.html

Dans app.js, j'ai déplacé cette route pour qu'elle ne soit pas surchargé par "blog/:id"

Le service Blog a aussi une nouvelle méthode pour sauver un billet

Dans le formulaire, on utiliser ng-model pour "binder" des champs du scope à un "input". Quand la vue change la valeur du champs, le scope est modifié. C'est le principe du "two way data-binding".

Le formulaire utilise ng-submit ce qui a pour effet d'attacher l'évènement de sousmission à l'appel de fonction. Cette fonction est attéchée au scope dans le controller (constollers/blogedit.js). Cette fonction appelle simplement la nouvelle méthode de service "Blog.save()"
